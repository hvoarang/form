<form>

Формы позволяют получать информацию от посетителей веб-страницы. Списки рассылки, контактные данные, профиль пользователя, логин и регистрация, комментарии в блогах, всё это реализуется при помощи форм и их элементов.

Формы - это «денежные страницы», благодаря которым интернет-магазины продают свои товары, а онлайн сервисы получают оплату за предоставленные услуги. Компании измеряют успех своего веб-сайта эффективностью его форм, потому что они отвечают на такие вопросы, как «сколько потенциальных клиентов наш веб-сайт отправил отделу продаж?» и «сколько людей подписалось на наш продукт на прошлой неделе?». Поэтому формы постоянно рефакторятся и улучшаются.

Функциональная HTML-форма состоит из двух частей: интерфейс пользователя и сервер (бэкэнд). Первое - это внешний вид формы (разметка и оформление), а второе - код обработки введённых данных (валидация, хранение данных, отправка электронной почты и т. д.).

Элемент <form>

Элемент <form> это контейнер для группы связанных элементов формы. Например форма отзыва, регистрации, оплаты заказа в интернет-магазине. Каждый из этих компонентов интерфейса состоит из группы интерактивных элементов сгруппированных формой.

index.html
<form name="signup_form" autocomplete="on" novalidate>
<label>
Email
<input type="email" name="email" />
</label>

<label>
Passwordbr
<input type="password" name="password" />
</label>

<button type="submit">Submit</button>
</form>

У элемента <form> нет обязательных атрибутов, поэтому в примере приведены несколько полезных необязательных.

  • name - уникальное на текущей веб-странице имя формы. Используется как на сервере так и на клиенте при работе с формой. Может содержать символы английского алфавита в любом регистре, цифры, подчёркивание и тире. В имени нельзя использовать пробел.
  • autocomplete - определяет, может ли браузер автоматически заполнять значения всех элементов формы. Имеет всего два значения off и on. Это поведение можно будет изменить для каждого элемента формы.
  • novalidate - атрибут-флаг, не имеет значения. Говорит браузеру не проверять валидность введённых данных при отправке формы. Если атрибут не указан, выполняется клиентская валидация.

При клике по <button type="submit"> или при нажатии клавиши Enter в любом поле формы, она «отправляется», что приводит к перезагрузке текущей страницы. Это поведение по умолчанию, которое можно будет изменить при помощи JavaScript.

Элемент <label>

Элемент <label> связывает описание (метку) с интерактивным элементом формы. Текст метки связан с элементом не только визуально, но и логически. Например, скринридер огласит текст метки, когда пользователь поставит фокус на связанный элемент. При клике по метке, связанный элемент автоматически получит эффект фокуса.

Если элемент формы, например поле ввода, вкладывается в <label>, связь создаётся браузером автоматически. Клик по тексту «Email» или «Password» поставит фокус в соотвествующее поле ввода.

index.html
<form>
<label>
Emailbr <input type="email" name="email" />
</label>

<label>
Password
<input type="password" name="password" />
</label>

<button type="submit">Submit</button>
</form>

Если элемент формы не вложен в <label>, необходимо явно связать их через атрибут id связываемого элемента и атрибут for метки.

index.html
<form>
<label for="user_email">Email</label>
<input type="email" name="email" id="user_email" />

<label for="user_password">Password</label>
<input type="password" name="password" id="user_password" />

<button type="submit">Submit</button>
</form>

Элемент <input>

Универсальный элемент для создания разнообразных полей ввода. Тип поля определяется атрибутом type, значение которого по умолчанию text - однострочное текстовое поле принимающее любые символы.

<input type="text" name="username" />

Концептуально тег <input /> представляет запись в словаре, который отправляется на сервер. Атрибут name определяет имя этой записи, а значение - это то, что ввёл пользователь. Имя поля должно быть уникальным внутри формы. Значение атрибута name также используется при работе с формой в JavaScript.

Атрибут autofocus

Поле ввода, которому задан этот атрибут, автоматически получит фокус при загрузке страницы, и в нём можно сразу набирать текст. По умолчанию атрибут не установлен ни для какого поля.

Так как фокус не может быть у нескольких элементов одновременно, атрибут задаётся полю, с которого пользователю стоит начать заполнять форму. Атрибут autofocus - логический, то есть без значения, указывается только название.

Атрибут placeholder

Позволяет отображать текст-подсказку о том, какие данные необходимо ввести в поле, когда элемент <input> пуст. Этот атрибут можно использовать в любом элементе формы, где есть текстовый ввод.

Почта и пароль

Значение атрибута type влияет на тип поля, набор символов, которые в него можно ввести, и последующую валидацию значения. Например, когда браузер смартфона видит поле с типом email, он отображает пользователю специальную клавиатуру для ввода электронной почты с легко доступным символом @. Браузер также может предоставить способ выбрать адрес электронной почты из адресной книги.

Если задать атрибуту type значение radio - инпут превратится в переключатель (радио-кнопку, radio button). Радио-кнопки всегда идут группами, что позволяет пользователю выбрать одно из множества предопределенных значений.

Каждой радио-кнопке в группе задаётся одинаковое значение атрибута name, иначе браузер не будет знать, что это группа.

В отличие от текстовых полей, в радио-кнопки нельзя вводить данные, поэтому каждой из них необходимо указать значение в атрибуте value. Это значение, которое будет передано на сервер когда пользователь отправит форму.

Логический атрибут checked указывает какой переключатель будет выбран (отмечен) по умолчанию. В группе радио-кнопок в состоянии checked может быть только один элемент.

Чекбоксы (флажки)

Флажки (чекбоксы, checkbox) похожи на переключатели, но позволяют выбирать произвольное количество значений, то есть многие из многих. Чекбоксов может быть целая группа, например выбор хобби, либо только один, например принятие пользовательского соглашения при регистрации.

Числа

Обычные текстовые поля принимают буквально всё: цифры, буквы, пробелы, знаки пунктуации и т. п. Для того, чтобы разрешить вводить только числа, с возможностью ограничить диапазон, используется тип поля number. При вводе данных в числовое поле браузер автоматически игнорирует все символы кроме цифр

<label>
Age
<input type="number" name="age" value="0" min="18" max="120" />
</label>

Конечно же, есть много чисел, которые не подходят для каждого типа числовых данных. Например, может быть необходимо запретить ввод слишком больших, либо отрицательных чисел. Эта проблема решается с помощью атрибутов min и max, определяющих границы ввода.

По умолчанию числовые поля принимают только целые числа. Это поведение можно изменить с помощью атрибута step, который указывает шаг изменения числа. По умолчанию значение шага равно 1.

Телефонные номера

Тип tel создаёт поле ввода телефонных номеров, которые могут быть представлены в самых разных форматах. В настоящее время единственная польза от применения поля типа tel состоит в отображении виртуальной клавиатуры для ввода телефонных номеров в мобильных браузерах, которая содержит цифры, но не буквы.

<form>
<label>
Phone number
<input type="tel" name="phone_number" />
</label>

<button type="submit">Submit</button>
</form>

Выбранное значение: 40

Браузеры не предоставляют никакой визуальной обратной информации об установленном значении. Это делается с помощью JavaScript и дополнительной разметки.

Дата и время

В браузерах есть встроенный календарь, в котором пользователь может выбрать требуемую дату и/или время. Атрибуты min и max позволяют установить минимальные и максимальные даты, при условии использования правильного формата.

Из-за различного внешнего вида в разных браузерах и сложности его изменения через CSS, вместо нативного календаря чаще всего используют готовые JavaScript-библиотеки.

Элемент <textarea>

Создаёт многострочное текстовое поле для ввода большого количества текста. Например, для обратной связи, комментария, поста в соцсетях, уточнения деталей заказа и т. п.

Атрибут rows устанавливает количество строк (высоту), а cols - колонок (ширину). На практике указывается только rows, а ширина элемента контролируется через CSS.

По умолчанию элемент <textarea> можно растягивать по горизонтали и вертикали. Для того чтобы контролировать возможность изменения размера пользователем, в CSS есть свойство resize.

resize: both | horizontal | vertical | none

Элемент <select>

Выпадающее меню это альтернатива радио-кнопкам, поскольку по умолчанию позволяет выбрать один из многих вариантов. Элемент <select> это раскрывающееся меню с атрибутом name, которое содержит набор элементов <option> с атрибутом value.

Текст внутри элемента <option> отображается пользователю, а значение атрибута value это то, что будет использовано при отправке формы.

По умолчанию выбран первый элемент <option> из списка. Это можно изменить задав необходимой опции атрибут-буль selected.

Элемент <datalist>

Автозаполнение - это приём с которым знакомы все пользователи. При вводе символов в текстовое поле, поисковая система предлагает различные варианты автозаполения. При наборе нового сообщения электронной почты, почтовый клиент предлагает список получателей и т. д.

Элемент <datalist> создает список предопределённых значений, которые можно выбирать при наборе в связанном текстовом поле. Этот список скрыт и становится доступным при получении инпутом фокуса и наборе текста в нём. Список, создаваемый элементом <datalist>, связывается с текстовым полем через атрибут id. Его значение должно совпадать со значением атрибута list тега <input>.

Элементы <option> также могут иметь атрибут value. Это полезно, когда текст опции и значение не совпадают. Например, пользователь может не знать сокращение штата в США. В выпадающем меню он увидит список штатов и их сокращений, а при выборе опции, текстовое поле будет заполнено только значением опции.

Группировка полей

Группировка связанных элементов делает формы более понятными для пользователей. Это также помогает пользователю сосредоточиться на небольших упорядоченных группах полей, а не пытаться понять всю форму сразу. Группировка выполняется как визуально в интерфейсе, так и логически в коде.

Элементы <fieldset> и <legend>

Элемент <fieldset> это контейнер для группировки нескольких связанных элементов в форме, а вложенный <legend> выполянет роль заголовка группы. Связанные радио-кнопки и чекбоксы всегда должны быть сгруппированы, другие типы полей группируются по необходимости.

Группировка с возможностью оформления

Элементы <fieldset> и <legend> имеют отличную семантику, но ограниченные возможности оформления. В большинстве случаев для группировки элементов формы используется <div>; с атрибутами доступности role и aria-labelledby.

<div role="group" aria-labelledby="contact-details-head">
<p id="contact-details-head">Enter your contact details</p>
Related elements
</div>

  • Атрибут role="group" указывает на то, что элементы внутри этого <div> являются частью группы.
  • Атрибут aria-labelledby="contact-details-head" содержит идентификатор элемента с описанием группы.
